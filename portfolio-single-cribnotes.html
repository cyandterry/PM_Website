Overview

CribNotes is a web service which is similar to CliffNotes and SparkNotes, which provide served as a resource to help users understand books, write papers, and study for tests. It provides book summaries, study guides and all kinds of resources.
However, one thing that differentiate CribNotes from others is that users can request specific books/journals/literature that they would like a summary of, and other users can bid to create the summary for the requester. The user requesting the summary accepts bids from different users, and then chooses the user he/she would like to create the summary. The users bidding to create the summary provide a bid price for creating a summary, and have a profile that provides examples of past work he/she has completed.

My Role

It is an exercise in which I need to design the whole service platform. It's very challenging to design the service all by myself in just a few days' time, which asks you to be very logical and consider steps of the users' interaction with the system. Especially this is a request and bidding service, which involves a community of users and their activities and interaction with each other.

  • Conducted research about CliffNotes and SparkNotes which have similar service as CribNotes
  • Designed task flow of the main services like book searching and bidding
  • Designed wireframes of main pages and interactions of the website
  • Created mockup for the homepage

The Process

Research

  • Examined the content, and every page, functions and interactions of CliffNotes and SparkNotes
  • Conducted research about other websites that have freelance or bidding service, such as eBay, Freelancer.com, and Guru.com

Design User Flow
I sketched down all my ideas along the process before I started to create the user flow.
  • Since the key feature of the website is that users can request and bid for summaries, which involves money transaction in the process. Thus, I thought it would be better to create a requester-bidder community and trustworthy platform. This asks for users to login with real identity and credit card information. For writers, they also need personal profile to show their past works and feedbacks from others.

  • Users can either search by books or search by bids. When searching by bids, they will be directed to the "Crib Market" page and all related bids will be shown.

  • To avoid causing confusion and wasting users' time on searching, the system will aggregate all the requests and bids for the same book together.

  • When users search for book summaries, the user-created summaries will appear in the search results too, along with the official one (if there's one) from CribNotes, but with obvious visual cues to differentiate them.
